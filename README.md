📄 Отчёт о выполнении задачи 1: парсинг данных парковок с 2GIS

1. Постановка задачи
Целью проекта было собрать структурированные данные о парковках в городе Алматы с платформы 2GIS, включая такие параметры как название, адрес, координаты, вместимость, тип парковки, способы оплаты и дополнительные характеристики.

2. Первичный подход: использование API 2GIS
В начале работы я решила использовать официальный API 2GIS, так как это наиболее корректный и стабильный способ получения данных.
Я зарегистрировалась и получила API-ключ, после чего начала тестировать запросы к поиску объектов.
Однако в процессе работы я столкнулась с ограничением — лимит запросов API был быстро исчерпан. Из-за этого получить полный набор данных через API оказалось невозможно, так как парковок много, а лимиты не позволяли сделать полную выгрузку.
➡️ Итог: данный подход был частично успешен, но не подошёл для полного сбора данных.

3. Переход к веб-скрейпингу (Selenium)
После ограничения API я перешла к альтернативному методу — веб-скрейпингу с использованием Selenium.
Я начала анализировать структуру сайта 2GIS и выяснила, что большая часть данных загружается динамически через HTML и встроенные JSON-структуры.
На этом этапе я написала первый рабочий скрипт, который:
собирал ссылки на страницы парковок через поиск
открывал каждую страницу автоматически
извлекал данные через page_source и body.text

4. Проблемы и доработки
В процессе разработки возникло несколько проблем:
❌ 1. Неполные или пустые данные
Некоторые поля (например, вместимость или рейтинг) часто возвращали None, так как данные не всегда присутствовали в текстовом виде.
❌ 2. Ошибки в адресах
Первоначально адреса извлекались из текста страницы, что приводило к неточным или неполным значениям.
❌ 3. Неверная структура данных
На ранних этапах были ошибки в названиях столбцов и логике формирования DataFrame, частично из-за автоматической генерации кода с помощью AI.

5. Улучшение подхода
После анализа структуры сайта я перешла к более надёжному методу извлечения данных — использованию JSON, встроенного в HTML страницы 2GIS.
Это позволило:
корректно извлекать координаты (POINT(lat lon))
получать вместимость из поля "capacity"
определять цену парковки (час/сутки)
извлекать типы услуг и оплат через атрибуты (attribute_groups)
Также я улучшила обработку признаков парковки:
охраняемая / крытая / тёплая
доступность для грузовиков
способы оплаты (наличные, карта, QR)
режим работы (24/7)

6. Финальная версия решения
В итоговой версии скрипта:
используется Selenium для навигации по сайту
собираются ссылки на все парковки
каждая страница парсится через page_source
данные извлекаются из JSON-структуры и HTML
результаты сохраняются в CSV файл через pandas
